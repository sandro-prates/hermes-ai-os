## 2. Arquitetura Atual

- Aplicação FastAPI em `apps/backend/app/main.py`.
- Router principal em `apps/backend/app/api/router.py`, com prefixo `/api/v1`.
- Health endpoint em `apps/backend/app/api/v1/health.py`.
- Configuração central baseada em `pydantic-settings` em
  `apps/backend/app/core/settings.py`.
- Observabilidade central em `app.core.observability`.
- Logging configurado com `logging.config.dictConfig` e namespace `hermes`.
- Formatters `ConsoleFormatter` e `JsonFormatter`.
- Middleware ASGI puro para logging HTTP e correlação de requisições.
- Contexto assíncrono de Request ID baseado em `ContextVar` e restauração por `Token`.
- Snapshot técnico baseado em projeção determinística da árvore Git, conforme ADR-0005.

Não há implementação funcional de banco de dados, runtime de agentes, memória,
dashboard ou integrações externas.

## 3. Funcionalidades Implementadas

- Importação e inicialização da aplicação FastAPI.
- `GET /` com metadados e estado da aplicação.
- `GET /api/v1/health` com status HTTP 200 e dados do ambiente.
- Settings centralizados para aplicação, servidor, logging e Request ID.
- Logging nos formatos `console` e `json` selecionado por `LOG_FORMAT`.
- Geração automática de `X-Request-ID` quando ausente.
- Preservação do Request ID enviado pelo cliente.
- Inclusão do Request ID no header da resposta.
- Logs HTTP de início, conclusão e falha com método, caminho, status e duração.
- Testes automatizados de contexto, formatters, middleware e gerador de snapshot.
- README operacional com instalação, execução, endpoints e comandos de qualidade.
- Gerador oficial executado por `python tools/project_snapshot.py`.
- Verificação determinística executada por `python tools/project_snapshot.py --check`.
